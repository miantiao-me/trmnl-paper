# 电子墨水屏设计规则

`trmnl-paper-takumi` 默认产出的是 **TRMNL 800x480 静态主视觉**，不是普通彩色 Web UI 截图。

风格关键词：**黑白、高对比、排版驱动、工业感、像素友好、长时间静态显示也成立**。

## 1. 先按电子墨水屏思考，再按 Web 画图

- 画面可能会停留数分钟到数小时，因此必须经得起长时间静置
- 默认优先白底黑字；大面积反白或整屏深底会更容易显脏、显重
- 动效、hover、scroll、玻璃拟态、阴影层次都不是这条路线的重点
- 想不清某个装饰是否该保留时，默认删掉

## 2. 三层视觉层级

每屏只保留三层重要性：

1. **主焦点**：唯一第一眼看到的元素，例如大号时间、一个核心数值、一句标题
2. **次级信息**：解释主焦点的上下文，例如标签、单位、副标题、次级数据
3. **三级元信息**：时间戳、状态、来源、设备信息等，放在边缘，不能抢主视觉

约束：

- 主焦点只能有一个，不要同时出现两个同等大的 hero
- 次级信息贴近主焦点组织；三级信息尽量推到边缘或底部
- 如果眯眼看图时分不出谁最重要，说明层级失败

## 3. 黑白与灰阶

- 默认只用 **纯黑 + 纯白** 建立主体结构
- 最多保留 **1 级辅助灰**，只用于弱化标签、次要线条、背景纹理
- 不要照搬 Nothing 风格里 4 档以上透明度层级；在墨水屏上容易发虚
- 连续灰阶照片不是默认答案；能改成块面、线稿、像素化或明确抖动时更稳
- wrapper 默认优先启用 `image-dither`；只有在原图已是纯黑白硬边块面且实机确认不加更好时，才关闭它

## 4. 排版预算

单屏默认预算：

- 默认最多 **2 个字体家族**；如果主焦点需要额外的 display 点阵体，可放宽到 **3 个**
- 最多 **3 个字号层级**
- 最多 **2 个字重**

优先级：

- 层级先靠字号差，再靠位置和留白
- 标签、元数据、数字补充信息优先用更克制的样式，可考虑 mono
- 动态数字、时间、计数器优先用 monospace 或 `tabular-nums`，避免位数变化时排版跳动
- 中文场景优先保证字形完整与清晰，不必强绑 Nothing 的原始字体栈
- 想加新字号前，先问自己能不能靠间距解决

可吸收的字体角色：

- **主数值 / 时间**：可用点阵风 display，但只建议用于 36px 以上 hero，不用于正文
- **正文 / 标题**：优先几何无衬线；中文场景始终以可读性优先
- **标签 / 数据 / 单位**：优先等宽字；建议 11–12px、全大写、`0.06em–0.1em` 字距
- **单位**：紧贴数值，使用 label 级小字号，而不是和主数值同重同大

具体字体栈与加载建议见 `takumi-basics.md`。

## 5. 留白与分组

间距本身就是语义：

- **4–8px**：强绑定，例如数值与单位
- **12–16px**：同组不同项
- **24–32px**：新分组开始
- **48px+**：制造主次反差与海报感

规则：

- 优先用留白分组，不要先画框
- 只有在同构密集数据里才考虑细分割线
- 避免所有块都用相同间距平铺，那会让画面失去主次
- 如果必须做视觉切分，优先 `1px` 实线边框 / 分隔线，不要用阴影
- 密集数据行优先用 divider，不要用 zebra striping
- 典型 stat row 可用：左侧小号等宽全大写标签，右侧数值，单位紧贴数值

## 6. 构图：非对称、贴边、给空白留位置

- 优先非对称构图，例如左大右小、上重下轻、边缘锚定
- 不要把所有元素都安全地缩在中心小盒子里
- 允许一大块留白存在，它本身就是视觉张力的一部分
- 居中只适合极少数场景，例如纯时钟、纯徽标、单个图标式主视觉

## 7. 像素、线条与纹理

- 像素素材必须整数倍缩放，并显式写 `imageRendering: "pixelated"`
- 避免半像素、亚像素、过细浅灰线
- 点阵、分段条、网格纹理可以用，但前提是缩小后仍然清晰
- 纹理只能辅助，不要压过主信息

推荐吸收的具体表达：

- **分段进度条**：优先方头矩形段，不要圆角胶囊；段与段之间保留 `1–2px` 间隙
- **图表**：优先线图 / 柱图；不要大面积 area fill，不要单独 legend 盒子，尽量直接标在线旁或配数值
- **辅助网格**：只保留最少量 `1px` 水平虚线 / 细线，帮助读数即可
- **点阵纹理**：可用 `1px` dot-grid + `12–16px` 节距做轻纹理，但只能在大留白或弱背景区域低强度使用

## 8. 数据组件倾向

- 进度、容量、完成度优先用 **分段式仪表条**，同时配一个明确数值
- 看板小组件优先“左上角分类标签 + 左对齐大号数字 + 紧邻单位”的机械式排布
- 状态表达优先靠黑块、空心、实线、虚线、纹理差异，不靠彩色状态灯
- 空态、加载态都要尽量静态化：文字提示、分段条、点阵占位比 skeleton 更适合

## 9. 可吸收的 Nothing 风格

这些原则适合吸收进 takumi：

- **Subtract, don’t add**：默认做减法
- **Typography as UI**：用排版而不是装饰建立层级
- **Data as beauty**：大号数字、等宽数据本身就是主视觉
- **Asymmetry & edge anchoring**：不追求保守对称，而是制造海报张力
- **Industrial tone**：保留理性、机械、精密的气质
- **No shadows**：依靠留白、边框、分割线建立深度
- **Dot-grid / segmented motifs**：允许把点阵与分段条当成少量视觉母题

## 10. 不要直接照搬的 Nothing 元素

- 不要把 **dark / light mode 双模式 token** 原样搬进来；TRMNL 默认仍以白底黑字更稳
- 不要依赖多级灰度 / 透明度做细层级
- 不要引入 Nothing 标志性的红色点缀；当前路线默认只有黑、白与极少量辅助灰
- 不要把机械开关、交互动效、悬停反馈当作核心表达
- 不要为了“像 Nothing”而强绑它的字体栈；当前 skill 仍以现实可用字体与中文兼容优先
- 不要照搬圆角卡片、按钮、输入框整套组件库；TRMNL image-first 更适合主视觉、看板块和静态数据构图

## 11. 出图前快速审查

至少自查这几项：

- 这张图是不是只有一个真正的主焦点？
- 是否主要依靠黑白对比，而不是多层灰度？
- 是否真的留出了空白，而不是把 800x480 填满？
- 标签、元信息是否已经降权并推到边缘？
- 像素素材是否使用整数缩放与 `pixelated`？
- 分段条、点阵、细线在缩小后是否仍然清晰？
- 如果去掉一个装饰元素，画面会不会更好？如果会，就删掉
